# 开物（Kaiwu）· 行业技术蒸馏术

> 天工开物，提炼行业技术框架

## 简介

开物（Kaiwu）是一个用于**行业技术蒸馏**的AI Skill框架。输入任意行业名称或技术领域，自动完成深度调研 → 技术框架提炼 → 生成可运行的行业技术Skill。

## 核心能力

### 🔍 多源信息采集
6个并行Agent分别调研：
- **技术栈** - 核心技术原理与架构
- **标准规范** - 行业标准与协议
- **术语体系** - 专业术语与概念定义
- **市场视角** - 竞争格局与技术路线分歧
- **技术演进** - 版本迭代与架构升级
- **发展时间线** - 行业里程碑与最新动态

### 🧠 技术框架提炼
- **技术框架**（3-7个）- 行业解决问题的核心模式
- **工程启发式**（5-10条）- 技术选型的快速规则
- **术语体系** - 专业表达范式与概念层级
- **技术价值与反模式** - 核心追求与明确避免
- **技术谱系** - 技术传承与影响脉络

### 📦 一键生成Skill
输出可直接运行的行业技术顾问Skill，包含：
- 完整的Agentic Protocol（回答工作流）
- 技术框架详解（含应用场景与局限性）
- 诚实边界标注（明确Skill覆盖范围）

## 使用方式

### 直接路径（明确行业）
```
用户：开物，蒸馏AI视频生成行业

开物：收到，开始调研AI视频生成行业...
[执行6个Agent并行调研]
[提炼技术框架]
[生成Skill]
```

### 诊断路径（模糊需求）
```
用户：我想了解现在最火的AI技术

开物：你说的"AI技术"是指：
1. AI生成技术（视频/图像/音频）
2. 大语言模型技术
3. AI芯片/硬件技术
4. 其他？

用户：主要是AI视频生成

开物：明白了，推荐蒸馏「AI视频生成行业」，核心技术包括扩散模型、自回归生成等...
```

## 特性

- ✅ **模糊需求诊断** - 自动推荐合适的行业/技术领域
- ✅ **本地语料优先** - 支持用户提供的白皮书/论文/报告
- ✅ **质量验证流程** - 确保技术框架的准确性
- ✅ **诚实边界标注** - 明确Skill的覆盖范围
- ✅ **自包含设计** - 生成的Skill独立运行，不依赖外部文件

## 安装与使用

### 作为WorkBuddy Skill使用

1. 将本仓库克隆到WorkBuddy skills目录：
```bash
cd ~/.workbuddy/skills
git clone https://github.com/Nightando/kaiwu-skill.git
```

2. 在WorkBuddy中触发：
```
开物，蒸馏[行业名称]
```

## 工作流程

```
Phase 0: 入口分流 → 明确行业 / 模糊需求诊断
Phase 0.5: 创建目录结构
Phase 1: 6个Agent并行调研
Phase 1.5: 调研Review检查点
Phase 2: 技术框架提炼
Phase 2.5: 提炼确认检查点
Phase 3: Skill构建
Phase 4: 质量验证
Phase 5: 双Agent精炼
```

## 质量验证标准

| 检查项 | 通过标准 |
|--------|---------|
| 技术框架数量 | 3-7个，每个有来源证据 |
| 每个框架的局限性 | 明确写出失效条件 |
| 术语体系辨识度 | 读100字能识别是哪个行业 |
| 诚实边界 | 至少3条具体局限 |
| 内在张力 | 至少2对技术矛盾 |
| 一手来源占比 | >50% |

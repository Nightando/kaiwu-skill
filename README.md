## ✨ 效果示例

### 示例1：AI视频生成行业咨询

```
用户      ❯ 现在做AI视频生成，应该选扩散模型还是自回归路线？

开物      ❯ 这不是二选一的问题，是资源约束下的技术路线赌博。

扩散模型（如Stable Diffusion、Midjourney）：
- 优势：图像质量高、可控性强、生态成熟
- 代价：推理慢、算力成本高、时序一致性难保证
- 适合：图像生成为主、对质量要求高的场景

自回归模型（如GPT-4o视频、VideoPoet）：
- 优势：原生视频理解、时序连贯性好、可扩展性强
- 代价：训练成本极高、开源生态薄弱
- 适合：长视频生成、需要理解视频内容的场景

我的判断：
如果你团队<10人、算力预算<100万，选扩散模型路线，用ComfyUI生态快速验证。
如果你有大厂资源或融资>5000万，可以赌自回归，但做好3年生态建设的准备。
```

### 示例2：半导体行业技术选型

```
用户      ❯ 28nm成熟制程和7nm先进制程，怎么选？

开物      ❯ 先回答一个问题：你的产品对功耗和面积的敏感度是多少？

如果是物联网芯片、电源管理、MCU：
- 28nm完全够用，成本低、产能充足、IP生态成熟
- 7nm对你来说是过度工程，多花3倍钱换1%的性能提升

如果是手机SoC、AI加速器、高性能计算：
- 7nm是入场券，不是选择题
- 但准备好面对：流片成本>5000万、良率爬坡6个月、EDA工具授权年费百万级

关键启发式：
「先进制程不是技术问题，是商业问题。选28nm还是7nm，取决于你的客户愿不愿意为那一点性能提升付10倍的钱。」
```

## 🎯 蒸馏了什么

开物不是复制行业报告，是**提炼行业技术框架**。

| 层次 | 说明 | 示例（AI视频生成） |
|------|------|-------------------|
| **技术框架** | 行业解决问题的核心模式 | 扩散模型、自回归生成、多模态融合 |
| **工程启发式** | 技术选型的快速规则 | 「如果团队<10人，选扩散模型路线」 |
| **术语体系** | 专业表达范式 | Latent Space、CFG Scale、时序一致性 |
| **技术反模式** | 行业明确避免的方案 | 忽视推理成本的产品化、过度追求SOTA |
| **诚实边界** | Skill覆盖不到的部分 | 不能预测全新技术路线的突破时间 |

**关键区分**：捕捉的是 HOW the industry works，不是 WHAT they built。

## 🔧 工作原理

```
Phase 0: 入口分流 → 明确行业 / 模糊需求诊断
    ↓
Phase 1: 6个Agent并行调研（技术栈/标准规范/术语体系/市场视角/技术演进/时间线）
    ↓
Phase 2: 技术框架提炼（三重验证：跨域复现、生成力、排他性）
    ↓
Phase 3: Skill构建（Agentic Protocol + 技术框架 + 工程启发式）
    ↓
Phase 4: 质量验证（已知测试 + 边缘测试 + 术语测试）
    ↓
Phase 5: 双Agent精炼（auto-skill-optimizer + skill-creator）
```

### 六路并行采集

| Agent | 采集目标 | 输出 |
|-------|---------|------|
| 1 技术栈 | 技术白皮书、架构文档、学术论文 | 核心技术与架构 |
| 2 标准规范 | 行业标准、协议文档、API规范 | 技术约束与标准演进 |
| 3 术语体系 | 技术博客、开发者文档、社区讨论 | 专业术语与表达范式 |
| 4 市场视角 | 行业报告、竞品分析、投资研报 | 市场格局与技术分歧 |
| 5 技术演进 | 版本发布、技术迭代、重大更新 | 技术决策与架构升级 |
| 6 时间线 | 行业诞生到现在的完整发展线 | 关键里程碑与最新动态 |

### 质量验证标准

| 检查项 | 通过标准 | 不通过信号 |
|--------|---------|-----------|
| 技术框架数量 | 3-7个，每个有来源证据 | <3或>10 |
| 每个框架的局限性 | 明确写出失效条件 | 只写优点 |
| 术语体系辨识度 | 读100字能识别是哪个行业 | 像通用技术文档 |
| 诚实边界 | 至少3条具体局限 | 只有「不能替代专家」 |
| 内在张力 | 至少2对技术矛盾 | 观点高度一致（太假） |
| 一手来源占比 | >50% | 主要依赖二手转述 |

- ✅ **模糊需求诊断** - 自动推荐合适的行业/技术领域

## 🎨 品牌故事

> 「天工开物，技术致知」

开物（Kaiwu）源自《天工开物》——中国第一部关于农业和手工业生产的综合性著作。

既然我们可以蒸馏人的思维方式，为什么不能蒸馏整个行业的技术逻辑？

开物造的不是术，是一张地图。一个好的行业技术Skill，让你用行业的视角看技术问题。不是为了照搬技术，而是为了理解技术背后的逻辑。
